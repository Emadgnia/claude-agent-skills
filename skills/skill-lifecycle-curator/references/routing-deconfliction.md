# Routing Deconfliction

Use this process when new skills overlap with existing ones.

1. Compare trigger phrases in all candidate descriptions.
2. Keep the narrowest skill for specialized workflows.
3. Keep one broad fallback skill for adjacent tasks.
4. Add explicit precedence in `Skills.md` when two skills can trigger.
5. Document why overlap is acceptable (or how it was removed).

Red flags:

- Two skills share the same primary trigger words.
- One skill repeatedly needs instructions from another to complete baseline tasks.
- Routing causes frequent manual overrides.
