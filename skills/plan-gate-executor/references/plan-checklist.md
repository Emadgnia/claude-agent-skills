# Plan Checklist

Use this checklist before requesting approval:

- [ ] Task name follows `YYYY-MM-DD-short-task-name.md`
- [ ] Goal explains intended outcome in 1-3 sentences
- [ ] Scope clearly separates in/out of scope
- [ ] Missing inputs listed as explicit questions
- [ ] Execution steps are ordered and testable
- [ ] Validation commands are concrete
- [ ] Risks include impact + mitigation
- [ ] Rollback includes trigger + verification
- [ ] Approval section includes requester + status + timestamp

Use this checklist after implementation:

- [ ] Plan status updated (`In Progress` -> `Done`)
- [ ] Validation results captured
- [ ] Documentation synced (`Skills.md`, `Architecture.md`, tests docs when needed)
- [ ] Residual risk and blockers recorded
