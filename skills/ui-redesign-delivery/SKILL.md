# UI Redesign Delivery

Use this skill for implementation-grade UI redesign work, not superficial styling tweaks.

## Workflow

1. Extract product goal, audience, and conversion priority.
2. Define visual direction (type, color system, spacing, motion).
3. Redesign information architecture and key flows.
4. Implement responsive UI with reusable primitives.
5. Validate accessibility, readability, and interaction clarity.

## Design Rules

- avoid default/boilerplate aesthetics
- choose intentional typography and color system
- include meaningful motion only where it supports hierarchy
- ensure desktop and mobile parity

## Delivery Artifacts

- redesigned component/page code
- design tokens or CSS variable system
- rationale for major visual decisions
- responsiveness and accessibility checks
